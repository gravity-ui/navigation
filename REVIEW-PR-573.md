## Summary

PR добавляет тень (gradient overlay) внизу `ScrollableWithScrollbar`, чтобы визуально сигнализировать пользователю о наличии скрытых элементов меню ниже видимой области.

---

## Замечания

### 1. [Critical] Хардкод `rgba(0, 0, 0, 0.08)` — сломано в тёмной теме

```scss
background: linear-gradient(to bottom, transparent 0%, rgba(0, 0, 0, 0.08) 100%);
```

Чёрная тень `rgba(0, 0, 0, ...)` на тёмном фоне будет **невидимой или почти невидимой**. В тёмной теме нужен другой подход — например, использование дизайн-токенов Gravity UI.

В этом же проекте в `HighlightedItem` и `Item` уже используется `var(--g-color-sfx-shadow)` — это правильный подход, адаптирующийся к теме.

**Рекомендация:** Использовать CSS custom property (например, `--gn-aside-shadow-color`) с дефолтом на дизайн-токен, либо напрямую использовать `var(--g-color-sfx-shadow)`. Это также позволит потребителям библиотеки кастомизировать тень.

### 2. [Medium] Нет тени сверху при прокрутке вниз

Тень показывается только снизу (`hasContentBelow`), но когда пользователь прокрутил вниз — контент скрыт и сверху. Для полноценного UX стоит добавить аналогичную верхнюю тень, когда `scrollState.scrollTop > 0`.

Если это осознанное решение — стоит обосновать в описании PR.

### 3. [Medium] Магическое число `52px` для высоты тени

```scss
height: 52px;
```

Значение `52px` захардкожено и ничем не обосновано. Вероятно, это высота одного пункта меню, но:
- Это не задокументировано и не очевидно.
- Если высота пункта меню изменится — тень будет выглядеть непропорционально.

**Рекомендация:** Вынести в CSS custom property или хотя бы добавить комментарий, поясняющий выбор значения.

### 4. [Low] Магическое число `-1` в условии

```tsx
scrollState.scrollTop + scrollState.clientHeight < scrollState.scrollHeight - 1;
```

`- 1` — это порог для компенсации дробных пикселей при субпиксельном рендеринге. Это корректный приём, но стоит:
- Вынести в именованную константу (например, `SCROLL_BOTTOM_THRESHOLD = 1`), чтобы было понятно назначение.

### 5. [Low] Отсутствие плавного появления/исчезновения тени

Тень появляется и исчезает мгновенно (conditional rendering через `{hasContentBelow && ...}`). Это может выглядеть как визуальный «скачок».

**Рекомендация:** Вместо условного рендеринга использовать `opacity: 0/1` с `transition: opacity 150ms ease`, а элемент рендерить всегда. Это даст плавное появление/исчезновение.

```scss
&__bottom-shadow {
    // ... existing styles ...
    opacity: 0;
    transition: opacity 150ms ease;

    &_visible {
        opacity: 1;
    }
}
```

```tsx
<div className={b('bottom-shadow', {visible: hasContentBelow})} aria-hidden />
```

### 6. [Low] Нет возможности отключить тень через props

Поведение тени включено безусловно. Для библиотечного компонента стоит рассмотреть возможность отключения через prop (например, `showShadow?: boolean`, по умолчанию `true`), чтобы потребители могли контролировать это поведение.

Это minor замечание — можно добавить позже при необходимости.

### 7. [Nit] `aria-hidden` без явного значения

```tsx
<div className={b('bottom-shadow')} aria-hidden />
```

В JSX `aria-hidden` без значения эквивалентно `aria-hidden=""`, что может интерпретироваться по-разному. Лучше явно указать:

```tsx
<div className={b('bottom-shadow')} aria-hidden="true" />
```

- Минимальный, сфокусированный PR — изменения затрагивают только нужные файлы.
